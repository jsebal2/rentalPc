# 개요
이 제품은 기존 PC 임대 업장에서 수기로 관리하던 엑셀 기반 업무를 대체하는 웹 기반 원격 PC 임대 관리 시스템입니다. 복잡한 대여 기간, 납부 상태, 고객 커뮤니케이션, 유지보수 등의 업무를 자동화하고, 데이터 기반의 운영을 가능하게 합니다. 대상은 PC 임대 사업자 및 관리자이며, 반복적인 수작업을 줄이고 운영 효율성을 높이는 것이 핵심 가치입니다.

# 핵심 기능

### 1. 대시보드 요약
- **기능:** 전체 시스템 현황(PC 사용 수, 미사용 수, 매출, 고객 수 등)을 실시간으로 요약 표시
- **중요성:** 관리자가 운영 상황을 직관적으로 파악 가능
- **구성 방식:** 각 모듈(PC, 사용자, 결제 등)의 데이터를 시각화하여 카드 및 그래프로 출력

### 2. PC 관리
- **기능:** PC 등록, 사양 수정, 사용 상태 추적 가능
- **중요성:** 자산 현황과 원격 접속 가능 장비 관리의 핵심
- **구성 방식:** CRUD 기반의 인터페이스와 태그형 상태 표시, 사용 기록 포함

### 3. 사용자 및 계약 관리
- **기능:** 사용자 정보 및 계약 기간, 자동 연장 여부 등 관리
- **중요성:** 고객과 장비를 연결하는 핵심 관리 체계
- **구성 방식:** 사용자와 렌탈 정보를 관계형으로 연결, 기간 설정 기능 포함

### 4. 결제 상태 추적
- **기능:** 월 납부 내역 기록, 미납/완납 구분, 자동 연장 여부 확인
- **중요성:** 수익 흐름 추적 및 고객 관리에 필수
- **구성 방식:** 사용자 계약과 결제 이력을 연동, 상태 태그로 표시

### 5. 알림 및 자동 경고
- **기능:** 계약 만료, 미납, 유지보수 필요 시 자동 문자/이메일 발송
- **중요성:** 반복 응대 자동화 및 누락 방지
- **구성 방식:** 조건 기반 이벤트 트리거 + 템플릿 메시지 시스템

### 6. 장애 및 수리 이력 관리
- **기능:** 장비 문제 발생 시 이력 저장 및 처리상태 기록
- **중요성:** 유지보수 일정 관리 및 장비 수명 예측 가능
- **구성 방식:** 사용자/PC 연동 + 시간별 로그 저장 + 상태 변경 로그

### 7. 관리자 및 권한 제어
- **기능:** 관리자 계정 별 권한 차등, 이력 기록
- **중요성:** 보안 강화 및 운영 책임 구분
- **구성 방식:** 권한별 접근 제한 + 작업 로그 저장

# 사용자 경험

### 사용자 유형
- **사업자/관리자:** 전체 운영 상황과 계약 관리가 필요한 사용자
- **직원:** 고객 응대 및 기술 지원 중심 사용자
- **고객(선택사항):** 로그인하여 대여 내역/요청 제출 가능 (MVP에는 포함 안 됨)

### 주요 플로우
1. 관리자 로그인 → 대시보드 확인 → 미납 고객 확인 → 문자 발송
2. 새 PC 등록 → 사용자 연결 → 계약 기간/결제 상태 설정
3. 계약 만료 도래 → 시스템 자동 알림 → 관리자 수동/자동 연장 처리

### UI/UX 설계 고려사항
- 테이블 중심 구성 + 빠른 필터 기능 제공
- 대시보드 카드/차트로 요약 정보 제공
- 모달 기반 등록 및 수정 → 페이지 전환 최소화

# 기술 아키텍처

### 시스템 구성요소
- 프론트엔드: Vue.js
- 백엔드: Node.js(Express)
- DB: MySQL 또는 PostgreSQL
- 알림 스케줄러 + 외부 문자 API 연동

### 데이터 모델
- Users, PCs, Rentals, Payments, MaintenanceLogs, Admins

### API 및 외부 연동
- REST API 기반 내부 통신
- 문자 발송 API (Twilio, 카카오비즈메시지 등)
- 추후: 결제 웹훅 연동 고려 가능

### 인프라 요구사항
- 클라우드 서버 (예: AWS EC2 + RDS)
- 보안 접속 (HTTPS), 관리자 대시보드 전용 로그인 필요

# 개발 로드맵

### Phase 1 (MVP)
- 관리자 로그인 + 대시보드
- PC 등록 및 수정 기능
- 사용자 및 계약 연결
- 수동 결제 처리 UI
- 수동 문자 발송 기능

### Phase 2
- 자동 알림 스케줄러 구현
- 장애 이력 관리 기능
- 대시보드 통계 추가
- CSV 및 PDF 내보내기 기능

### Phase 3
- 관리자 권한 제어 세분화
- 계약서 파일 첨부 기능
- 고객 요청 티켓 시스템
- 고객 전용 로그인 (사용 내역 확인)

# 논리적 개발 순서

1. **기초:** 로그인/인증 + 데이터모델 구축 (User/PC/Rental)
2. **1차 UI:** PC 관리, 사용자 관리, 계약 등록 페이지 완성
3. **가시적 완성:** 결제/문자 기능 추가로 운영 가능 수준 확보
4. **기능 확장:** 자동화 시스템, 수리이력, 데이터 내보내기 기능 추가
5. **선택 확장:** 고객 로그인, 실시간 통계, 고급 알림

# 리스크 및 대응 방안

### 기술적 리스크
- 문자 발송 실패 등 → 신뢰도 높은 API 연동 + 전송 로그 저장
- 관계형 모델 오류 방지 → FK 강제 및 폼 유효성 검사 적용

### MVP 범위 정의
- MVP는 "관리자 중심 기능만" 구축
- 고객 계정 기능은 추후 확장

### 자원 제약
- 각 기능을 모듈화 → PC / 사용자 / 결제 독립 개발 가능
- 오픈소스 UI/차트 라이브러리 적극 활용

# 부록

### 현장 조사 요약
- 대부분 수기/엑셀 관리 중 → 오류, 누락, 속도 문제 빈번
- 자동 문자 기능에 대한 수요 매우 높음

### 기술 사양 요약
- MySQL 기반 관계형 스키마
- React + REST API 구조
- AWS 기반 배포 가정
